Session Token mit Web Url holen

Ein Session Token gilt für {{ .Duration}} und muss dann erneuert werden.

Dem angegebenen User wird über das System eine Berechtigung (permission) zugewiesen.

Der User bzw das Password ist nicht zwingend das Password für die angegebene Datenbank. Diese werden über das REST System verwaltet.

Wird kein User angegeben wird 'gast' als Default gesetzt.

  • http://localhost:4488/api/token/get?Database=D:/Data/DokuMents/DOKUMENTS30.FDB
  • http://localhost:4488/api/token/get?Database=D:/Data/DokuMents/DOKUMENTS30.FDB&Location:192.168.11.98&Port:3050&Password:superuser&User:su
  • http://seserver.ddnss.de:4488/api/token/get?ftext=Database=D:/Data/DokuMents/DOKUMENTS30.FDB&Location=Localhost
  • http://seserver.ddnss.de:4488/api/token/get?fjson={"Database":"D:/Data/DokuMents/DOKUMENTS30.FDB"},{"Location":"Localhost"}

    • Session Token mit curl holen

  • curl -X GET -H "Content-Type: application/json" http://seserver.ddnss.de:4488/api/token/get?ftext="Database=D:/Data/DokuMents/DOKUMENTS30.FDB&Location=Localhost"
  • curl -X GET -H "Content-Type: application/json" http://seserver.ddnss.de:4488/api/token/get?fjson="{"Database":"D:/Data/DokuMents/DOKUMENTS30.FDB},{"Location":"Localhost"}"

    Beispiel Returnvalue: {"Status":200,"Message":"Created UUID, permissions:All, duration:30m0s","Data":"{{.Key}}"} }

    SQL Anweisungen mit Sessionkey

  • http://localhost:4488/{{.Key}}/db/sql/get?cmd=SELECT * FROM TSTANDORT
  • http://localhost:4488/{{.Key}}/db/sql/get?cmd=SELECT * FROM TSTANDORT&info=Info zum command
  • http://localhost:4488/{{.Key}}/db/sql/get?ftext="cmd=SELECT FROM TSTANDORT"
  • http://localhost:4488/{{.Key}}/db/sql/get?fjson="{"cmd":"SELECT FROM TSTANDORT"}"

    • Der entsprechende SQL Befehl kann 'maskiert' werden indem das Maskierungszeichen der SQL Anweisung vorangestellt wird.

    Beispiel: http://localhost:4488/{{.Key}}/db/sql/get?q=cmd:'SELECT'*'FROM'TSTANDORT

    REST GET mit Fields,Filter,Order,Group

    Mögliche Parameter

  • fields -> fields=(id,bez) , fields=(*)
  • order -> order=(id asc,bez desc)
  • group -> group=(id,bez)
  • filter -> filter=(BEZ like 'T%')
  • limit -> limit=(10) , limit=(10,100)

  • http://localhost:4488/{{.Key}}/rest/get/TSTANDORT?fields=(id,bez)&order=(bez asc, id desc)&filter=(BEZ like 'T%')&group=(id,bez)
  • http://localhost:4488/{{.Key}}/rest/get/TSTANDORT?ftext="fields=(id,bez)&order=(bez asc, id desc)&filter=(BEZ like 'T%')&group=(id,bez)"
  • http://localhost:4488/{{.Key}}/rest/get/TSTANDORT?fjson="{"fields":["id","bez"]},{"order":["bez asc","id desc"]},{"filter":"BEZ like 'T%'"},{"group":["id","bez"]}"

    • SQL Anweisungen Testen ohne Sessionkey

  • http://localhost:4488/api/sql/test?ftext="cmd=SELECT * FROM TSTANDORT"

    • Der entsprechende SQL Befehl kann 'maskiert' werden indem das Maskierungszeichen der SQL Anweisung vorangestellt wird.
